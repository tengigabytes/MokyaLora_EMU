# 00 — 設計憲章與全域規則

## 設計憲章

| 原則 | 內容 |
|---|---|
| **焦點唯一** | 全系統任一時刻僅有一個橙色焦點框 |
| **三層上限** | Launcher 桌面 → App → Detail/Modal；設定樹例外，用麵包屑+折疊壓縮深度感 |
| **模式即 App** | Meshtastic 功能群以九個獨立 App 承載，避免單一 App 內 Tab 爆炸 |
| **文字輸入是子模式** | D-pad 在 IME 中改變語意，由 IME 接管直到 BACK 退出 |
| **全域反射** | Status Bar 永駐，Hint Bar 僅在子模式顯示（設定 App 例外，全程不顯示） |
| **對齊 Meshtastic** | 設定項命名與層級對齊官方 protobuf，便於對照文件除錯 |

## 顯示骨架（320×240）

```
┌────────────────────────────────────────┐
│ STATUS BAR             16px            │
├────────────────────────────────────────┤
│                                        │
│         APP CONTENT  208px             │
│                                        │
├────────────────────────────────────────┤
│ HINT BAR               16px (條件顯示)  │
└────────────────────────────────────────┘
```

- **Status Bar**：永駐頂端，所有頁面繼承
- **App Content**：各頁自定義
- **Hint Bar**：僅在子模式（編輯、彈窗、特殊狀態）顯示；桌面與九宮格不顯示；設定 App 全程不顯示
- **IME Bar**：模式 D 文字輸入有候選字時顯示在最底端 18px

### 配色語意

| 視覺 | 含義 |
|---|---|
| 主色白 #E6EDF3 | 預設、正常、Op 態 |
| 綠 #39D353 | 良好、TX 完成、未讀計數、3D Fix |
| 黃 #F1E05A | 注意、2D Fix、警告 |
| 紅 #F85149 | 嚴重、無 GPS、低電、SOS |
| 橙 #FFA657 | TX 動作、IME 輸入態、焦點（D-pad 行為已變） |
| 灰 #30363D | 不動作、停用、無資料 |

**核心視覺原則**：**任何橙色都意味「D-pad 行為已切換」**——使用者掃一眼螢幕看到橙色就知道目前 ←→ 是游標移動、選項選擇等特殊操作，而非翻頁或一般導航。

## 字型策略

**全程使用 GNU Unifont 16×16 點陣字**。

| 用途 | 規格 |
|---|---|
| CJK 字元 | 16×16 點陣 |
| ASCII / 半形 | 8×16 點陣 |
| 行高（一般） | 22px（16px 字 + 上下 3px padding） |
| 行高（緊湊） | 20px（16px 字 + 上下 2px padding） |
| 行高（標題） | 24px |
| 字數計數、單位 | 12px ASCII（特例） |

### Unifont 子集容量估算

- BIG5 常用字 5,401 + 次常用字 7,652 = 13K 字
- ASCII 95 + 標點符號 200 = ~300 字
- 合計 ~13.3K 字 × 32 bytes = **約 425 KB**
- 索引表 ~30 KB
- **總計 ~460 KB flash**

選擇 Unifont 的理由：
1. 涵蓋全 Unicode BMP（不會缺字）
2. 點陣字實機可讀性穩定
3. 字型管理單純
4. 開源（GPL/OFL）

縮小到 14px 不可行：Unifont 是純點陣字，縮放會失真，繁體字筆畫密集（鬱、龜、靈）會糊成一團。

## 全域行為慣例

| 情境 | 行為 |
|---|---|
| 收到新 DM | Status Bar `✉` 計數變化 + 短提示音；不打斷當前操作 |
| 收到 SOS 廣播 | 全屏 alert 覆蓋；OK 確認後回原 App，事件留在桌面節點事件區 |
| LoRa TX 中 | Status Bar `▲` 持續亮，禁止重複發送 |
| 低電量 <15% | Status Bar 電量轉黃 |
| 低電量 <5% | 桌面切極簡低電模式 + Status Bar 整條警告態 + 系統默默省電 |
| 低電量 <2% | 螢幕關閉，任意鍵亮屏 5s |
| GPS 失定位 >5 分鐘 | Status Bar GPS 圖示變紅 + 警告燈亮起 |
| 無操作 30 秒 | 背光降至 50% |
| 無操作 2 分鐘 | 螢幕關，LoRa 仍收，任意鍵喚醒 |

## 警告態優先序

```
SOS 啟動 (整條 Status Bar 覆蓋)
  > 收到 SOS (整條覆蓋)
  > 極低電量 <5% (整條覆蓋)
  > 系統警告 (僅 ⚠ 燈)
  > 正常 Status Bar
```

## 焦點模型

**全系統任一時刻僅有一個焦點**，以橙色 #FFA657 邊框或 `▶` 標記表示。焦點所在區決定 D-pad 語意。

焦點不會落在純顯示元素上（如 Status Bar、儀表行）。

## 退出策略

| 操作 | 行為 |
|---|---|
| BACK 短按 | 上一層 |
| BACK 在桌面 | 無作用（桌面是最頂層） |
| BACK 在文字輸入 | 模式 A：取消變更 / 模式 B：存草稿 |
| BACK 長按 | 鎖屏 |
| 30 秒無操作 | 焦點回主要區段第 1 筆（不主動退出 App） |

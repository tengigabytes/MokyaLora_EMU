# 20 — L-0 桌面 / Home（Launcher 主頁）

## 角色定位

桌面承擔三重身份：

1. **狀態儀表**：上半部，純顯示，不可選
2. **通知中樞**：下半部，分為「訊息」與「節點事件」兩個獨立區塊
3. **快速入口**：FUNC 鍵呼出九宮格 / OK 進入當前焦點對話

## 通知雙軌分離

桌面下半部分為**兩個獨立區段**：

```
▼ 收件                          3 未讀     ← 訊息區（DM + 頻道訊息）
  [訊息列表]
─────────────────────────────────────
◎ 節點事件                      2 則       ← 事件區（上線/離線/位置請求等）
  [事件列表]
```

兩區共用 D-pad 上下移動焦點，焦點可跨區段流動。

**為什麼分開**：訊息有「對話延續性」，節點事件是「一次性通知」。兩者操作期望不同——訊息是「進去回覆」，節點事件是「知道了就過」。混在一起會讓清空提醒的邏輯混亂。

**節點事件範例**：
- `Tony 上線了`
- `阿明 離線`（>5 分鐘未廣播）
- `老周 請求你的位置`
- `新節點加入：!def67890`
- `KX9-Repeater 位置更新（移動了 1.2km）`

## 各區塊規格

### 第 1 行：身份 + 日期（20px）

```
洛克 !abc12345              04-28 We
```

| 元素 | 規格 |
|---|---|
| 暱稱 | Long Name 截 6 全形字內，超過 `…`。主色 `#E6EDF3` |
| Node ID | `!` + 8 碼 hex。次色 `#7D8590` |
| 日期 | 西元 `mm-dd We`（We=星期英文縮寫 Mo Tu We Th Fr Sa Su）。次色 |

### 第 2 行：GPS（20px）

```
◉ 24.149°N 120.681°E ±3m 3D
```

| 圖示 | 含義 | 顏色 |
|---|---|---|
| `◉` | 3D Fix | 綠 |
| `◉` | 2D Fix | 黃 |
| `◌` | 搜尋中 | 灰 |
| `✕` | 無定位/逾時 | 紅 |

- 座標格式依設定（十進位度 / 度分秒 / UTM / TWD97）
- 精度 `±3m` 來自 HDOP × 5
- 文字標 `3D` / `2D` / `搜尋` / `無定位`

### 第 3 行：電源（20px）

```
▣87% 4.02V -180mA  ~11h24m
```

| 圖示 | 含義 |
|---|---|
| `▣` | 放電中（電池底色填充） |
| `⚡` | 充電中 |
| `▪` | 已滿且仍接電源 |
| `!` | 異常（過熱/過壓），整體紅色 |

| 顏色 | 條件 |
|---|---|
| 主色 | >30% |
| 黃 | 15-30% |
| 紅 | 5-15% |
| 紅閃爍 | <5%（觸發極簡低電模式） |

- 電壓：2 位小數
- 電流：負放電 / 正充電，過去 30s 平均
- 預估剩餘到 15% 門檻；充電中改 `~2h12m to 100%`；待機 `──`

### 第 4 行：環境（20px）

```
🌡28.4° 📊1013.2→ ⛰2,847m
```

- **溫度**：感測器讀值，1 位小數
- **氣壓 + 趨勢**：QNH 海平面氣壓（依 GNSS 海拔自動換算） + 3h 趨勢
  - `→` 變化 < ±0.5 hPa（穩定）
  - `↗` 上升 > +0.5 hPa（天氣轉好）
  - `↘` 下降 > -0.5 hPa（天氣轉壞，登山警訊）
  - `⇈` 急升 > +1.5 hPa/3h
  - `⇊` 急降 > -1.5 hPa/3h（鋒面接近，山難前兆）
- **海拔**：GNSS 優先；無 GNSS 改 `⛰~2,847m`（波浪號表氣壓海拔）
- **氣壓更新頻率**：每 10 分鐘

### 第 5 行：網路（20px）

```
☷ChU18% Tx4% H:2m Hop2.1 SNR-8
```

| 指標 | 顏色門檻 |
|---|---|
| ChU（Channel Utilization） | <25% 綠 / 25-40% 黃 / >40% 紅 |
| Tx（本機 air time） | <10% 綠 / 10-25% 黃 / >25% 紅 |
| H:Xm（LastHeard） | <5m 綠 / 5-15m 黃 / >15m 紅 / >60m 顯示「離線」 |
| Hop | 最近 10 筆封包平均跳數 |
| SNR | >-10 綠 / -10~-15 黃 / <-15 紅 |

### 訊息區段

**標頭（18px）**：
```
▼ 收件                          3 未讀
```

- `▼` 暗示往下捲還有訊息（按 D-pad 下到底自動進 A-1 訊息 App）
- 未讀數 ≥1 時主色綠 `#39D353`，全已讀時不顯示數字

**條目（20px，每筆 40 字位）**：

```
▶👤阿明     09:38 ●2 我快到了五分鐘有
```

| 區塊 | 字位 | 內容 |
|---|---|---|
| 焦點符 ▶ | 2 | 留空時 2 半形空白 |
| 圖示 | 2 | `👤` DM / `#` 頻道 / `📢` 廣播 / `⚠` SOS（紅置頂） |
| 名稱 | 8 | 4 全形字（截斷符號 `…`） |
| 空白 | 1 | |
| 時間 | 5 | 今日 `HH:MM` / 昨日 `昨日` / 更早 `mm-dd` |
| 空白 | 1 | |
| 未讀標 | 3 | `●N` 主色綠 1全+1半 |
| 空白 | 1 | |
| 預覽 | 17 | 約 8 全形字，已讀者灰色 |
| **合計** | **40** | 剛好填滿 |

### 節點事件區段

**標頭（18px）**：
```
◎ 節點事件                     1 則
```

**條目（20px）**：

```
↑ Tony 上線                    09:30
```

| 符號 | 含義 | 顏色 |
|---|---|---|
| `↑` | 上線 | 綠 |
| `↓` | 離線（>15 分鐘未聽到） | 灰 |
| `?` | 請求位置 | 藍 |
| `+` | 新節點首次加入 | 主色 |
| `○` | 節點移動超過 500m | 黃 |
| `⚠` | 對方發 SOS | 紅 |

事件區滿後舊事件自動退場（FIFO）。

## 互動行為

### D-pad 行為

| 操作 | 焦點所在 | 行為 |
|---|---|---|
| ↑ | 訊息第 1 筆 | 無作用 |
| ↑ | 訊息其他筆 | 上移一筆 |
| ↑ | 事件第 1 筆 | 跳到訊息最後一筆 |
| ↓ | 訊息中間筆 | 下移一筆 |
| ↓ | 訊息最後一筆 | 跳到事件第 1 筆（若事件區有內容） |
| ↓ | 事件最後一筆 | 進入訊息 App（A-1） |
| → 或 OK | 訊息焦點 | 進入該對話（A-2） |
| → 或 OK | 事件焦點 | 進入該節點詳情（C-2） |
| ← | 訊息焦點 | 移除該訊息提醒（unread→0，不刪內容） |
| ← | 事件焦點 | 移除該事件 |
| FUNC | 任意 | 呼出九宮格功能表 |
| BACK | 任意 | 無作用 |
| 長按 BACK | 任意 | 鎖屏 |
| 長按 Power 5s | 任意 | 啟動 SOS |
| 注音鍵 | 任意 | 無作用（字首快跳已移除） |

### 焦點預設

- 開機 / 喚醒：焦點 = 訊息區第 1 筆
- 訊息為空但事件有：焦點 = 事件區第 1 筆
- 兩區皆空：無焦點，唯一可用是 FUNC
- 30 秒無操作：焦點回訊息區第 1 筆
- 收到新訊息：頂端插入，焦點若已在第 1 筆則跟著上移；其他位置不動

## 實作備忘

### 重繪頻率

- 1 Hz 全更新時間/電量
- 5 Hz 局部更新（TX/RX 指示燈閃爍）
- 即時更新訊息區（收到新訊息）
- 30 秒更新環境/網路指標

### Framebuffer

320×240×16bpp = 150KB，雙緩衝 300KB，放 PSRAM。

### 字型

- 標題與儀表 16px Unifont CJK
- ASCII 8×16 Unifont
- 數據與單位可考慮 12px ASCII 點陣字（仍待決定，先用 8×16）

### 配色

依 `00-design-charter.md` 全域 token，桌面不引入新色票。
